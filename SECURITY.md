# Security Policy

## Reporting a Vulnerability

If you find a security vulnerability in this website, please report it responsibly. **Do not open a public issue.**

Instead:

1. Email us at **[yucheng.lee29@gmail.com]** with the details, or
2. Use GitHub's [private vulnerability reporting](../../security/advisories/new) on this repo

Please include:

- A description of the vulnerability
- Steps to reproduce it
- The affected page/URL or file
- Potential impact (if known)

## What to Expect

- We'll acknowledge your report within **72 hours**
- We'll investigate and aim to fix valid issues within **14 days**
- We'll credit you in the fix (unless you prefer to stay anonymous)

## Scope

In scope:

- XSS, injection, or other client-side vulnerabilities
- Exposed secrets, API keys, or credentials
- Authentication/login bypass issues
- Anything that could compromise visitor data

Out of scope:

- Issues on third-party services we don't control
- Automated scanner reports with no proof of concept
- Social engineering / phishing

## Supported Versions

Only the latest deployed version of the site is supported. Older commits/deployments are not maintained.

Thanks for helping keep this site secure! 🔒
